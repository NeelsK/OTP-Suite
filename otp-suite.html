Hide Pad in Image

Embeds one or more pads into a cover image using LSB (Least Significant Bit) steganography. The pad data is first encrypted with AES-256-GCM, then hidden in the pixel values. The output image looks identical to the original.

Important: Save and share the output as PNG only. JPEG or other lossy formats will destroy the hidden data.

Extract Pad from Image

Load a stego image and enter the passphrase to recover the hidden pads. The extracted pads are loaded directly into your pad library.

Character Encoding Table

Each character is mapped to a 2-digit code. During encryption, each digit of the code is added (mod 10) to a digit from the pad. Decryption reverses this with subtraction (mod 10).

How It Works

Encryption: For each character in the message, look up its 2-digit code. Then add each digit to the next unused pad digit, modulo 10. The result is the ciphertext.

Decryption: For each pair of ciphertext digits, subtract the corresponding pad digit (mod 10) to recover the original 2-digit code, then look up the character.

Example: Letter "H" = code 07. Pad digits are 5 and 3. Ciphertext: (0+5)%10=5, (7+3)%10=0 → "50".

Security rules: Never reuse any portion of a pad. The pad must be at least as long as the message (in digit-pairs). Destroy pads after use. Share pads only through a secure, separate channel.
